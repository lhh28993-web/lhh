## 功能改进建议

### 概述
为 `stop-local-site.cmd` 脚本增加日志输出功能，以便更好地追踪和调试脚本的执行过程。

### 当前状况
当前脚本运行时，用户无法看到具体的执行结果或错误信息（输出已被重定向到 `/nul`）。

### 建议改进
1. **添加执行日志** - 记录脚本的运行时间和操作结果
2. **显示进程信息** - 显示是否找到了 3000 端口上的进程
3. **错误提示** - 当没有找到进程时给出明确提示
4. **可选的日志文件** - 支持将日志写入文件供后续查阅

### 示例输出
```
[2026-03-17 10:30:45] 开始检查 3000 端口...
[2026-03-17 10:30:45] 找到进程 PID: 12345
[2026-03-17 10:30:46] 成功终止进程
[2026-03-17 10:30:46] 操作完成
```
